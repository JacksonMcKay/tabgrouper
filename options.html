How to write rules

One rule per line.

Rules follow the format PATTERN TAB_GROUP_NAME TAB_GROUP_COLOR.

Color is optional. Valid colors: grey, blue, red, yellow, green, pink, purple, cyan. Invalid colors are treated as if you didn't specify a color.

Patterns follow a similar format to that used for basic adblock rules. See The AdGuard filter documentation on basic rules for more details.

If no special characters are given, the pattern must appear somewhere in the URL

A pipe (|) at the beginning or end means that the URL must start or end with the following/preceding text (in other words, the pipe symbol anchors the pattern to the beginning/end)

A double pipe (||) at the beginning means the following text is the domain name. With this you don't have to specify http:// or https:// or any subdomains at the beginning in order to match.

If you just want to match a domain, putting || at the beginning is probably what you want

Asterisks (*) are wildcards and match any text

Carets (^) are separator characters, and can be used in place of a slash (/) to match URL separators. It also matches the end of the URL as well as other special characters such as ?, # or :.

Lines beginning with ! or # are comments, and blank lines are skipped.

Example rules

! Group all google sites together
||google.com Google green

! Group example.com sites together
||example.com Example grey
